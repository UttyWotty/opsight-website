The Problem

AI coding assistants have full access to your development environment

⚠

Secret Exposure

AI can read .env files, credentials.json, SSH keys, API tokens, and cloud configs. One wrong move and secrets end up in logs or commits.

✘

Destructive Commands

Nothing stops the AI from running rm -rf, DROP TABLE, git push --force, or chmod 777. One hallucination away from data loss.

⇆

Data Exfiltration

AI can curl files to external servers, base64-encode secrets, write to /tmp, or pipe data through netcat. No audit trail.

∅

Zero Visibility

Security teams have no way to see what the AI accessed, what commands it ran, or what it tried to do. Compliance auditors ask questions you cannot answer.

Three Enforcement Layers

Defense in depth, not a single point of failure

1

Behavioral Rules

18 mandatory security rules the AI reads at session start. Covers sensitive files, credentials, SQL safety, XSS, PII, and scope boundaries.

What Gets Blocked

🔒

Sensitive Files

.env, credentials, certificates, SSH keys, cloud configs, database connection strings, secrets directories.

⛔

Dangerous Commands

rm -rf, sudo, chmod 777, kill -9, disk ops, pipe-to-shell, DROP TABLE, TRUNCATE, DELETE without WHERE.

🚫

Data Exfiltration

curl/wget uploads, base64 encoding of secrets, /tmp writes, clipboard exfiltration, netcat channels.

📦

Untrusted Packages

pip/npm/gem from git URLs, custom registries, or direct downloads. Standard registries allowed.

🌐

Environment Escape

SSH/SCP, docker run/exec/build, production env vars, destructive terraform/kubectl.

👤

PII Leakage

Social Security Numbers, credit card numbers, Korean Resident Registration Numbers written into code.
